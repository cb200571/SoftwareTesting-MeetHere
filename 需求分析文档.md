# MeetHere 数据看板功能 — 需求分析文档

## 一、项目背景

MeetHere 是一个场馆预约管理系统，原有功能包括用户管理、场馆管理、订单管理、新闻管理、留言管理等模块。管理员在日常运营中需要了解平台的整体运营状况，但原有系统只能通过逐个查看列表来获取信息，效率较低。

为解决这一痛点，团队新增了**管理员数据看板**功能，让管理员能够一眼掌握平台运营数据。

---

## 二、用户需求

### 2.1 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| 管理员 | 系统后台管理人员（user表中isadmin=1） | 登录后台后可查看数据看板 |

### 2.2 需求描述

管理员登录后台后，需要一个数据看板页面来快速了解：

1. **平台整体数据**：总订单数、总营收、总用户数、总场馆数
2. **订单状态分布**：待审核、已通过、已完成、已拒绝的订单各有多少
3. **订单趋势变化**：按月/周/天查看订单量的变化趋势
4. **热门场馆排行**：哪些场馆最受用户欢迎

### 2.3 筛选需求

管理员需要能够按以下维度筛选数据：

- **日期范围**：选择开始日期和结束日期
- **场馆**：选择特定场馆或查看全部场馆
- **订单状态**：选择特定状态或查看全部状态
- **时间粒度**：按月、按周、按天查看趋势

## 三、功能需求

### 3.1 功能列表

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| F-01 | 查看概览数据 | 展示4个数字卡片：总订单数、总营收、总用户数、总场馆数 | 高 |
| F-02 | 查看订单状态分布 | 以环形饼图展示4种状态的订单数量 | 高 |
| F-03 | 查看时间趋势 | 以折线图展示订单量的变化趋势，支持月/周/天切换 | 高 |
| F-04 | 查看热门场馆排行 | 以柱状图展示订单量TOP5的场馆 | 高 |
| F-05 | 日期筛选 | 选择开始日期和结束日期筛选数据 | 中 |
| F-06 | 场馆筛选 | 选择特定场馆筛选数据 | 中 |
| F-07 | 状态筛选 | 选择特定订单状态筛选数据 | 中 |
| F-08 | 时间粒度切换 | 切换按月/按周/按天查看趋势 | 中 |
| F-09 | 重置筛选 | 清空所有筛选条件，恢复默认 | 低 |

### 3.3 用例说明

| 用例编号 | 用例名称 | 前置条件 | 基本流程 | 后置条件 |
|---------|---------|---------|---------|---------|
| UC-1 | 查看数据看板 | 管理员已登录 | 1. 点击左侧菜单"数据统计"<br>2. 系统加载统计页面<br>3. 前端自动发出AJAX请求<br>4. 后端查询数据库返回JSON<br>5. 前端渲染数字卡片和图表 | 页面展示完整的数据看板 |
| UC-1.1 | 查看概览数据 | 进入数据看板页面 | 1. 前端请求 /statistics/overview.do<br>2. 后端统计订单、营收、用户、场馆数据<br>3. 返回JSON，前端渲染4个数字卡片 | 展示4个概览指标 |
| UC-1.2 | 查看订单状态分布 | 进入数据看板页面 | 1. 前端请求 /statistics/overview.do<br>2. 后端统计4种状态各自的订单数<br>3. 返回JSON，ECharts渲染环形饼图 | 展示订单状态饼图 |
| UC-1.3 | 查看时间趋势 | 进入数据看板页面 | 1. 前端请求 /statistics/monthlyOrders.do<br>2. 后端按时间粒度分组统计订单数<br>3. 返回JSON，ECharts渲染折线图 | 展示时间趋势折线图 |
| UC-1.4 | 查看热门场馆排行 | 进入数据看板页面 | 1. 前端请求 /statistics/venueRank.do<br>2. 后端统计订单量TOP5场馆<br>3. 返回JSON，ECharts渲染柱状图 | 展示TOP5场馆柱状图 |
| UC-1.5 | 筛选数据 | 进入数据看板页面 | 1. 管理员选择筛选条件<br>2. 点击"查询"按钮<br>3. 前端重新请求数据<br>4. 后端根据条件查询<br>5. 前端更新所有图表 | 图表展示筛选后的数据 |

## 四、非功能需求

### 4.1 性能需求

- 页面加载时间不超过3秒
- 图表渲染流畅，无卡顿
- 筛选操作响应时间不超过1秒

### 4.2 兼容性需求

- 支持Chrome、Firefox、Edge等主流浏览器
- 支持不同屏幕分辨率，图表自适应

### 4.3 安全需求

- 只有管理员（isadmin=1）才能访问数据看板
- 复用现有Session认证机制

## 五、数据需求

### 5.1 数据来源

| 数据 | 来源表 | 统计方式 |
|------|--------|---------|
| 总订单数 | order表 | COUNT(*) |
| 总营收 | order表 | SUM(total)，条件：state=3（已完成） |
| 总用户数 | user表 | COUNT(*) |
| 总场馆数 | venue表 | COUNT(*) |
| 订单状态分布 | order表 | GROUP BY state |
| 时间趋势 | order表 | GROUP BY DATE_FORMAT(order_time) |
| 热门场馆 | order表 | GROUP BY venueID ORDER BY COUNT(*) DESC LIMIT 5 |

### 5.2 筛选条件

| 筛选维度 | 参数名 | 默认值 | 说明 |
|---------|--------|--------|------|
| 开始日期 | startDate | 1970-01-01 | 格式：YYYY-MM-DD |
| 结束日期 | endDate | 2099-12-31 | 格式：YYYY-MM-DD |
| 场馆ID | venueId | 0 | 0表示全部场馆 |
| 订单状态 | state | 0 | 0表示全部状态 |
| 时间粒度 | granularity | month | 可选：month/week/day |

## 七、接口需求

### 7.1 接口列表

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 获取概览数据 | GET | /statistics/overview.do | 返回订单数、营收、用户数、场馆数、4种状态数量 |
| 获取时间趋势 | GET | /statistics/monthlyOrders.do | 返回按时间粒度分组的订单数量 |
| 获取场馆排行 | GET | /statistics/venueRank.do | 返回订单量TOP5场馆 |
| 获取场馆列表 | GET | /statistics/venueList.do | 返回所有场馆，用于下拉框 |

### 7.2 接口参数

所有接口支持以下筛选参数：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| startDate | String | 否 | 1970-01-01 | 开始日期 |
| endDate | String | 否 | 2099-12-31 | 结束日期 |
| venueId | int | 否 | 0 | 场馆ID，0表示全部 |
| state | int | 否 | 0 | 订单状态，0表示全部 |
| granularity | String | 否 | month | 时间粒度：month/week/day |
